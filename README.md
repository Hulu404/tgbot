# Бот заявок на техсопровождение

Telegram-бот для приёма заявок на техническое сопровождение мероприятий.
Бот по очереди задаёт вопросы организатору, формирует итоговую заявку,
отправляет её в рабочий чат и создаёт событие в Google Calendar.

## Возможности

- Пошаговый диалог (`ConversationHandler`) с валидацией даты и времени
- Подтверждение заявки перед отправкой (кнопки «Отправить» / «Заполнить заново»)
- Отправка заявки в рабочий Telegram-чат
- Автоматическое создание события в Google Calendar
- Устойчивость к сбоям: если календарь недоступен, заявка всё равно
  уходит в чат, а пользователь получает уведомление об ошибке календаря
- Команды `/start`, `/cancel`, `/help`

## Структура проекта

```
bot/
├── main.py          # точка входа, запуск бота
├── handlers.py      # ConversationHandler, все шаги диалога
├── calendar_api.py  # создание события в Google Calendar
├── config.py        # загрузка переменных из .env
├── .env.example     # шаблон переменных окружения
└── requirements.txt
```

## Установка и запуск

### 1. Зависимости

```bash
cd bot
python -m venv .venv
# Windows (PowerShell):
.venv\Scripts\Activate.ps1
# Linux / macOS:
source .venv/bin/activate

pip install -r requirements.txt
```

### 2. Конфигурация

Скопируйте шаблон и заполните значения:

```bash
# Windows (PowerShell):
copy .env.example .env
# Linux / macOS:
cp .env.example .env
```

Переменные в `.env`:

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен бота от BotFather |
| `WORK_CHAT_ID` | ID рабочего чата, куда падают заявки |
| `CALENDAR_ID` | ID Google-календаря (можно оставить пустым — тогда события не создаются) |
| `TIMEZONE` | Часовой пояс, по умолчанию `Europe/Moscow` |
| `GOOGLE_CREDENTIALS_JSON` | Путь к JSON-ключу сервисного аккаунта |

### 3. Запуск

```bash
python main.py
```

## Получение данных для `.env`

### 1. `BOT_TOKEN` — токен бота (BotFather)

1. Откройте в Telegram [@BotFather](https://t.me/BotFather).
2. Отправьте `/newbot`.
3. Введите имя бота и уникальный username (заканчивается на `bot`).
4. BotFather пришлёт строку вида `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` — это и есть `BOT_TOKEN`.

### 2. `WORK_CHAT_ID` — ID рабочего чата

Это чат (группа), куда бот будет присылать готовые заявки.

1. Добавьте вашего бота в нужную группу (а если нужны заявки в личку —
   просто напишите боту сам).
2. Самый простой способ узнать ID: добавьте в группу бота
   [@getmyid_bot](https://t.me/getmyid_bot) или [@RawDataBot](https://t.me/RawDataBot)
   — он покажет `chat id`.
3. Для групп ID обычно отрицательный, например `-1001234567890`.
   Для супергрупп он начинается с `-100`.

> Бот должен состоять в этом чате, иначе отправка не пройдёт.
> Если включён Privacy Mode, дайте боту админ-права в группе или отключите
> Privacy Mode через BotFather (`/setprivacy`).

### 3. Google Cloud: проект, Calendar API и `credentials.json`

Бот использует **сервисный аккаунт** (server-to-server, без ручного входа).

1. Откройте [Google Cloud Console](https://console.cloud.google.com/).
2. Создайте новый проект (вверху, селектор проектов → **New Project**).
3. Включите Calendar API:
   **APIs & Services → Library → Google Calendar API → Enable**.
4. Создайте сервисный аккаунт:
   **APIs & Services → Credentials → Create Credentials → Service account**.
   Задайте имя, роль можно не указывать. Нажмите **Done**.
5. Создайте ключ:
   откройте созданный сервисный аккаунт → вкладка **Keys** →
   **Add Key → Create new key → JSON**. Скачается файл — переименуйте его
   в `credentials.json` и положите в папку `bot/` (или укажите путь в
   `GOOGLE_CREDENTIALS_JSON`).
6. Скопируйте e-mail сервисного аккаунта (вида
   `имя@проект.iam.gserviceaccount.com`) — он понадобится в следующем шаге.

### 4. `CALENDAR_ID` и доступ к календарю

1. Откройте [Google Calendar](https://calendar.google.com/).
2. Выберите нужный календарь → **Настройки и общий доступ**
   (Settings and sharing).
3. В разделе **Предоставить доступ пользователям** (Share with specific people)
   добавьте e-mail сервисного аккаунта из шага 3.6 с правами
   **«Внесение изменений в мероприятия»** (Make changes to events).
4. В разделе **Интеграция календаря** (Integrate calendar) скопируйте
   **Идентификатор календаря** (Calendar ID) — это и есть `CALENDAR_ID`.
   Для основного календаря это ваш Gmail-адрес; для отдельного календаря —
   строка вида `...@group.calendar.google.com`.

> Без шага 4.3 (расшаривания календаря сервисному аккаунту) события
> создаваться не будут — это самая частая причина ошибки.

## Сценарий диалога

Бот последовательно спрашивает:

1. ФИО организатора
2. Контакт (телефон / Telegram / ВКонтакте)
3. Название мероприятия
4. Подразделение
5. Дата (ДД.ММ.ГГГГ)
6. Время занятости (ЧЧ:ММ–ЧЧ:ММ)
7. Площадка
8. Описание техсопровождения
9. Наличие ноутбука у организаторов

Затем показывает итоговую заявку и предлагает отправить её или заполнить
заново. После подтверждения заявка уходит в рабочий чат и добавляется в
календарь.
